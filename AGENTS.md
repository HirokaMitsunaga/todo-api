# Repository Instructions

このリポジトリで作業するときは、常に以下を守ること。

## 基本方針

- 実装前に要件・既存コード・設定・テストを確認する。
- 依頼に関係しないファイルは変更しない。
- 既存の設計・命名規則・実装方針を尊重する。
- 要件が曖昧な場合は、仕様を推測して拡張せず、前提や確認事項を整理する。

## 実装

- 必要最小限の変更で要件を満たす。
- 依存ライブラリは勝手に追加しない。追加が必要な場合は理由を説明する。
- 新しい設計やアーキテクチャへ置き換えない。

## 検証

- 実装後は可能な限り型チェック、Lint、テストを実行する。
- 実行できない場合は理由を報告する。

## 報告

- 変更内容
- 設計上の判断理由
- 実施した検証内容
- 残っている課題や懸念点

を最後にまとめて報告する。

繰り返し利用する実装フローは `.codex/skills/feature-implementation/SKILL.md` を参照すること。
